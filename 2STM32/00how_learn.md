### 🔹 2. **嵌入式 C 与你熟悉的 C++ 的区别**

面试官可能会问：“C 和 C++ 在嵌入式中有什么不同？”

✅ 关键点：

| 项目     | 嵌入式 C                                  | 你的 C++ 经验                                      |
| -------- | ----------------------------------------- | -------------------------------------------------- |
| 动态内存 | 避免 malloc/free（用静态分配）            | 你可能常用 new/delete → 要说明“我知道嵌入式要避免” |

| 标准库   | printf 可能不可用（需重定向到串口）       | 你知道 iostream，但嵌入式用串口打印                |

💡 面试可说：

### 🔹 3. **通信协议（重点：CAN + Modbus）**

#### ✅ CAN 总线（汽车/工业常用）

- 物理层：差分信号（抗干扰）
- 数据帧：ID（标识符）+ 数据（0~8 字节）
- 用途：ECU 之间通信（如电机控制器 ↔ 主控）

#### ✅ Modbus（工业经典）

- 基于 RS485（物理层）
- 协议简单：功能码（03 读寄存器，06 写单寄存器）
- 主从架构：主机轮询从机

📚 快速了解：

- 搜索：“CAN 总线工作原理 动图”
- 看一篇 Modbus RTU 报文示例（如 `01 03 00 00 00 01 84 0A`）

💡 面试可说：

> “我了解 CAN 用于高可靠实时通信，Modbus 是工业标准协议，常用于仪表数据采集。我有信心快速掌握具体实现。”

------

### 🔹 4. **RTOS vs 裸机（Bare-metal）**

- **裸机**：main() + while(1) + 中断 → 简单设备
- **RTOS**（如 FreeRTOS）：多任务、信号量、队列 → 复杂逻辑

✅ 了解 FreeRTOS 三个核心概念：

1. **Task（任务）**：独立运行的函数
2. **Queue（队列）**：任务间传数据
3. **Semaphore（信号量）**：资源保护

📚 推荐：看 FreeRTOS 官网 [Getting Started](https://www.freertos.org/)
